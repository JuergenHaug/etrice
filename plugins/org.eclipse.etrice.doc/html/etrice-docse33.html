7.3.1.1 DataLogging

Runtime logger for data-driven Messages with primtive data.

  • Annotation of one DataLogging per SubSystemClass
  • Specifiy complete port instance path starting from LogicalSystem
  • Port have to conjugated and data-driven
  • Activates logging of all messages having primitive or enum typed data
  • Seperate multiple port instance path through commas
  • Within pathlist do NOT use any whitespace (e.g. space or newline)

Features:

Uses:   ∙  Annotation

Similar to Java annotations they can be used to tag ROOM classes for further custom processing.

7.3.2 GenerationOptions

Mechanism to adjust the generation.

Options for generation are configured in the launch configuration or in case of standalone generation via command line. A list of available options:

  • generate as library
  • generate documentation
  • generate instrumentation for MSC generation
  • override output directories
  • some debug options

7.3.4 MSCLogging

Runtime logger for event-driven Messages, represented as a Message Sequence Chart.

The MSCLogging is activated by default, but can be set manually in the GenerationOptions. The output file is created upon regular termination of the application. The resulting file can be found in the logging directory and has the name ’msc.seq’, which can be open with the free open source tool Trace2UML (http://trace2uml.stage.tigris.org/)
